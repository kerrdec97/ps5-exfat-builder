Requirements & Installation

What you need

RequirementWhere to get itNotes

PS5 game dumpYour own PS5 game backupsMust contain eboot.bin
⚠️ The app must run as Administrator. It will prompt you automatically when you launch it.

First launch

  1. Download exFAT Image Builder.exe from the GitHub releases page
  2. Double-click the exe — Windows will ask for Administrator permission, click Yes
  3. If OSFMount is not found, a yellow banner appears at the top. Go to Settings → OSFMount and click Browse to point it to your osfmount.com file
  4. Set your Output directory — this is where your finished .exfat images will be saved
💡 Tip — OSFMount not detected?
  • Go to the Settings tab (⚙ in the tab bar)
  • Under OSFMount, click Browse
  • Navigate to where you installed OSFMount and select osfmount.com
  • It saves automatically — you won't need to do this again

Step-by-step: Building a single game

  1. Click Browse next to "Game root folder" and select your PS5 game folder. The app automatically reads the game title, PPSA ID and version from inside the folder
  2. Set your Output directory — where you want the finished .exfat saved (must have enough free space, same size as the game)
  3. The Output filename is filled in automatically e.g. ASTRO BOT (01.007.000).exfat
  4. Click + Add to Queue
  5. Click Build All to start building. A confirmation shows the estimated time before starting
  6. Watch the progress bar and file counter update in real time. When done the image is verified automatically

Building multiple games at once

Repeat steps 1–4 for each game to add them all to the queue, then click Build All. The app builds them one by one and notifies you when all are done.

💡 Tips
  • Use Save queue to save your game list as a .json file and reload it next time
  • The Recent dropdown remembers your last 10 game folders
  • The temp folder needs free space equal to the game size — if your C: drive is low, change it in Settings
  • Right-click a queue item to open the source folder, rebuild it, or remove it

📚 Library Tab — Managing Your Game Collection

The Library tab scans folders for PS5 game dumps and shows them as a visual collection with cover art. From here you can browse your games and add them to the build queue.

Setting up your library

  1. Click + Add folder and select the folder that contains your PS5 game dumps (e.g. D:\Games)
  2. You can add multiple folders — one per drive if you have games spread across drives
  3. Click 🔄 Scan to find all games. It searches for folders containing eboot.bin
  4. Games appear as cards with cover art, PPSA ID, version and size

Grid vs List view

Use the ⊞⊞ Grid and ☰ List toggle buttons to switch between views. Grid shows cover art cards. List shows compact rows — better for large collections.

Adding games to the build queue

Click any game card to open a context menu with options:

  • ➕ Add to Build Queue — adds the game immediately and switches to Build tab
  • 📋 View details — opens full info panel with output directory picker
  • 📂 Open folder — opens the game folder in Explorer

To build your entire library at once, click 🔨 Build All in the top bar — it adds every scanned game to the queue and switches to the Build tab.

🧩 Backports Tab — Applying Patches

The Backports tab lets you apply community backport patches to a game folder or mounted exFAT image. Backports allow newer games to run on older firmware versions.

Applying a backport to a game folder

  1. Select Game folder (on PC) as the target type
  2. Click Browse and select your PS5 game folder. The file list clears automatically
  3. Click + Add folder and select the backport folder you downloaded (e.g. Chapter 1 Backport 4.xx By Baderlink). The full folder structure is preserved automatically
  4. Choose your backup option — Next to target (auto) is recommended. It creates a backup folder named e.g. "Wuchang - Backport Backup (2026-04-25 15.26)"
  5. Click 🧩 Apply Backport
  6. A confirmation shows exactly which files will be overwritten and their sizes. Click Yes to proceed

Applying to an exFAT image

  1. Select exFAT image (mounted) as the target type
  2. Browse to your .exfat file and click 🔗 Mount — the image is mounted read-write
  3. Add your backport files and click Apply Backport as above
  4. When done, click 🔓 Dismount — the image is saved and ready
✅ You can also drag and drop a backport folder directly onto the file list — no need to click Add folder.
⚠️ Always keep a backup of original files. If the backport doesn't work you'll need to restore them.

📦 Payload Manager — Sending Payloads to PS5

The Payload Manager saves your .elf and .bin payload files and sends them to your jailbroken PS5 over a TCP network connection with one click.

Adding a payload

  1. Click + Add payload in the top right
  2. Click Browse and select your .elf or .bin file — the name is filled in automatically from the filename
  3. Add optional notes (e.g. "Use with firmware 4.xx")
  4. Click Save — it appears in your list and is remembered permanently

Sending a payload

  1. Enter your PS5 IP address and port at the top
  2. Click Save to remember the connection
  3. Click 📤 Send on any payload in the list
  4. The status bar shows live send progress and speed
PortUse
9090Common homebrew default
9021Alternative homebrew port
💡 Each payload shows a coloured badge — BIN or ELF. If a file is missing or moved the filename shows in red as a warning.

📡 FTP Upload — Sending Images to PS5

Once you've built your .exfat image you need to transfer it to your PS5 over FTP. Make sure your PS5's FTP server is running in your homebrew settings first.

Single file upload

  1. Go to Settings → PS5 FTP and enter your PS5's IP address and port (typically 2121 or 2122). Click Save
  2. Go to the 📡 FTP Upload tab
  3. Browse to your .exfat file
  4. Set the remote path on your PS5 (e.g. /data/etaHEN/games/)
  5. Click Upload. Progress, speed and ETA are shown live

Batch upload from My Images

Go to 💾 My Images, add your images folder, hold Ctrl+Click to select multiple images, then click ↑ Upload selected to PS5.

💡 Finding your PS5 IP
  • On your PS5: Settings → Network → View Connection Status
  • Or use Auto-detect PS5 IP in Settings — it scans your network automatically

📡 PS5 Browser — Managing Files on PS5

The PS5 Browser is a full FTP file manager that lets you navigate, rename, move, delete and download files directly on your PS5.

Connecting

  1. Click 🔗 Connect — uses the IP from your Settings
  2. Navigate folders by double-clicking, or type a path and press Enter
  3. Use ↑ Up to go up a folder level

Available actions

Delete🗑 Delete button or Delete key. Folders deleted recursively

New folder📂 New folder button
Refresh🔄 button or F5
⚠️ Move to... lets you type any destination path — the app creates the path automatically if it doesn't exist.

🗂 File Manager — Editing Images

The File Manager mounts a .exfat image as a Windows drive letter so you can browse and edit its contents without rebuilding.

  1. Browse to your .exfat file and click 🔗 Mount
  2. The image appears as a drive letter. Browse files and folders inside it
  3. Use the toolbar to Add files, Add folder, Replace, Delete or create New folder
  4. When done click 🔓 Dismount — changes are saved to the image
💡 This is useful for applying backports, replacing individual files, or adding extra content without a full rebuild.

  2. Click ▶ Connect
  3. Logs stream in real time with colour coding — 🔴 errors, 🟡 warnings, ⚫ debug, 🟢 info
  4. Use the Filter box to show only lines containing a keyword

🔧 Troubleshooting

ProblemFix
OSFMount not detectedGo to Settings → OSFMount → Browse and point it to osfmount.com manually
App freezes when browsing USB driveFixed in v1.4.2+. Update to the latest version
Image not dismounting after buildFixed in v1.4.2+. If it persists, eject manually from Explorer
Build failed — eboot.bin not foundYour game folder structure is wrong. eboot.bin must be in the root of the folder
Output drive low on spaceThe output drive needs free space equal to the game size. Use a bigger drive
FTP won't connectMake sure PS5 FTP server is running in your homebrew settings. Check IP and port
App crashes on startupA crash log is saved to C:\Users\[you]\exfat_builder_logs\. Share the log on GitHub Issues
Game name not detectedThe game folder may be missing param.sfo. The filename will default to the folder name
Build queue disappearsUse Save queue before closing so you can reload it next time
💡 If the app crashes, it automatically offers to copy a full error report to your clipboard. Paste it when opening an issue at github.com/kerrdec97/ps5-exfat-builder/issues
